## ✅ Major Changes (18/06/26) — Booking Structure Overhaul

### 🔒 Access Control — Non-member/Guest disabled

| การเปลี่ยนแปลง | รายละเอียด |
|---|---|
| `POST /bookings` ย้ายไป protected | ทุกคนต้อง login (`auth:sanctum`) ก่อนจอง |
| ลบ `POST /bookings/lookup` | Public lookup booking ถูกลบทิ้ง |
| ลบ `POST /bookings/{id}/request-payment` | Guest request payment link ถูกลบทิ้ง |
| `lookupBooking()` ลบ | ออกจาก `PaymentController` |
| `requestPaymentForGuest()` ลบ | ออกจาก `PaymentController` |
| Draft prevention เปลี่ยน | จาก `guest_email` → `user_id` (เพราะทุกคนต้อง login แล้ว) |

### 🚶 Walk-in ใช้ Staff User

| การเปลี่ยนแปลง | รายละเอียด |
|---|---|
| `FrontDeskController::walkIn()` | ไม่สร้าง guest user อีกต่อไป — ใช้ `verified_by` (staff/admin ID) เป็น `user_id` ของ booking |
| ข้อมูลผู้เข้าพัก | เก็บใน `booking_rooms.guests` (JSON) แทนการสร้าง user record |
| Payload ใหม่ | `verified_by`, `room_id`, `nights`, `guests[]`, `children` |

### 👥 Guest Data ย้ายไป `booking_rooms`

| การเปลี่ยนแปลง | รายละเอียด |
|---|---|
| ลบ columns จาก `bookings` | `guest_title`, `guest_name`, `guest_email`, `guest_phone`, `guest_nationality`, `is_ku_member`, `children` |
| เพิ่ม columns ใน `booking_rooms` | `guests` (JSON array — รองรับหลายคนต่อห้อง), `children` (integer) |
| Guest JSON structure | แต่ละ guest: `title`, `name`, `nationality`, `is_ku_member` |
| `Booking::primary_guest_name` accessor | ดึงชื่อแขกคนแรกจาก `bookingRooms.guests` JSON (fallback: user name หรือ "Customer") |
| `Booking` model | เก็บเฉพาะ `user_id` (who booked) — ไม่มี guest fields แล้ว |

## ✅ Fixed Issues

### #16: Guest สร้าง Draft Booking ซ้ำไม่จำกัด

**ปัญหา:** `BookingController::createBooking()` เช็ค active draft เฉพาะ authenticated user (`user_id`) เท่านั้น — Guest ที่ไม่ล็อกอินสามารถสร้าง draft booking ซ้ำกี่ครั้งก็ได้ด้วย email เดียวกัน ทำให้ระบบมี draft ขยะเต็มฐานข้อมูล

**วิธีแก้:** เพิ่ม `elseif` branch ใน `BookingController::createBooking()` ตรวจสอบ `guest_email` สำหรับ unauthenticated user — ถ้า email นั้นมี active draft (status=draft + payment_deadline ยังไม่หมด) อยู่แล้ว → สร้างใหม่ไม่ได้ คืน 422

**ไฟล์:** `BookingController.php:126-134`

**Tests:** `test_guest_cannot_create_booking_with_active_draft`, `test_guest_can_create_booking_after_draft_expired`, `test_guest_different_email_can_create_booking_independently`

### #6: Public Booking Routes ไม่มี Rate Limiting

**ปัญหา:** Routes `POST /bookings`, `POST /bookings/lookup`, `POST /bookings/{id}/request-payment` เป็น public (ไม่ต้อง login) แต่ไม่มี rate limiting — สแปมเมอร์สามารถยิงสร้าง draft booking หรือค้นหา booking ไม่จำกัด ทำให้ server โหลดหนักและฐานข้อมูลเต็ม

**วิธีแก้:** เพิ่ม middleware `throttle:5,1` (5 ครั้ง/นาที) ให้ `POST /bookings` และ `POST /bookings/{id}/request-payment`, และ `throttle:10,1` (10 ครั้ง/นาที) ให้ `POST /bookings/lookup` ใน `routes/api.php`

**ไฟล์:** `routes/api.php`

**Tests:** `test_create_booking_route_has_rate_limiting`, `test_booking_lookup_route_has_rate_limiting`, `test_guest_payment_request_route_has_rate_limiting`

### #15: Register ยอมรับ Role Field → Admin Escalation

**ปัญหา:** `AuthController::register()` ส่ง `$validated` ทั้งหมดเข้า `User::create()` รวมถึง `role`, `ver`, `is_ku_member` — ใครก็ตามสามารถสมัครแล้วตั้งตัวเองเป็น admin ได้ทันที โดยส่ง `role=admin` มากับ request

**วิธีแก้:** เปลี่ยน `register()` ให้เลือกเฉพาะ safe fields (`name`, `email`, `password`) สำหรับ `User::create()` — `role` จะใช้ default `'user'` จาก migration, `ver` และ `is_ku_member` ใช้ default `false`

**ไฟล์:** `AuthController.php:19-24`

**Tests:** `test_register_ignores_role_field_prevents_admin_escalation`

### #14: Booking Search Inconsistent กับ Booking List

**ปัญหา:** มีสอง route แยกกัน — `GET /bookings/search` (admin-only) แยกจาก `GET /bookings` (authenticated) ทำให้ duplicate logic, inconsistent security, และ test ใช้ `?q=` แต่ controller อ่าน `?term=` → search ไม่ทำงาน

**วิธีแก้:** Merge `bookingSearch()` เข้าไปใน `getBookings()` — ทุกคนใช้ `GET /bookings?term=...` route เดียวกัน: admin เห็นทุก booking + search ได้, user ทั่วไปเห็นแค่ของตัวเอง, ลบ route `GET /bookings/search` ทิ้ง

**ไฟล์:** `BookingController.php`, `routes/api.php`

**Tests:** `test_admin_can_search_bookings_with_term`, `test_admin_can_list_all_bookings`

### #13: DailyRoomMaintenance Log แสดงสถานะผิด

**ปัญหา:** ใน `DailyRoomMaintenance` command อ่าน `$room->status` หลังจากเรียก `transitionStatusTo('dirty')` แล้ว → log แสดง `status was 'dirty'` ทุกครั้ง แทนที่จะแสดงสถานะเดิมก่อนเปลี่ยน (เช่น `checkout_makeup`)

**วิธีแก้:** เก็บ `$originalStatus = $room->status` ก่อนเรียก `transitionStatusTo()` แล้วใช้ `$originalStatus` ใน log message เพื่อแสดงสถานะก่อนเปลี่ยน

**ไฟล์:** `DailyRoomMaintenance.php:79-82`
